3.2.1
Module Description
This module controls how institutional users are identified and authorized before
they can access protected parking functions. It relies on HCMUT_SSO to verify
user identity and retrieves authoritative profile data from HCMUT_DATACORE
in order to determine the user’s role and applicable parking privilege. Based on
the active parking policy, the system either grants or denies access to the requested
function. The module also records authentication and authorization outcomes to
support traceability, auditing, and secure operation across the entire Smart Parking
System.

3.3.1
Module Description
This module manages the entry-side lifecycle of a parking session for both insti-
tutional users and temporary users. It supports the beginning of a parking session
when a vehicle arrives at the entrance, validates the entry request, handles tempo-
rary ticket issuance for visitors or exceptional cases, and assigns an available parking
slot or target zone based on the current capacity state. If the selected parking area
no longer has valid capacity, the module denies entry and stops the creation of a new
session. In this way, the module helps reduce congestion at the gate, keeps session
data consistent from the beginning, and ensures that the parking area is used within
its available capacity.

1. The administrator opens the audit trail interface.
2. The system displays the most recent audit entries in reverse chronological
order.
3. The administrator applies filters such as date range, change type (policy
update, manual override, privilege change), or responsible actor.
4. The system retrieves and displays the matching audit records.
5. The administrator selects an audit entry to view its complete details, includ-
ing the previous configuration state and the new state after the change.
